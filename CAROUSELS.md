# PULSE — Логика трёх каруселей

> Официальная спецификация. Какая карусель что показывает.
> Последнее обновление: 2025-07-10

---

## Общий принцип

| # | Название | Что показывает | Для кого | Sentiment-карточки | Статус |
|---|----------|---------------|----------|-------------------|--------|
| 1 | **"Это вы ещё не видели"** | Непрочитанные новости по тегам пользователя | Только залогиненным | Liquid glass + Tag Impact pills | ✅ Работает |
| 2 | **"Вся лента"** | Прочитанные новости по тегам (хронология) | Только залогиненным | Liquid glass + Tag Impact pills | ✅ Работает |
| 3 | **"Общая лента"** | Все новости **с тегами** (без фильтра по пользовательским тегам) | Всем, включая без логина | Liquid glass + Tag Impact pills | ✅ Работает |

**Логика:**
- 1-я карусель = **новое** (ещё не видел) → landscape 425×225px
- 2-я карусель = **история** (уже видел, можно вернуться) → portrait 275px
- 3-я карусель = **полная лента** (всё что есть в базе) → portrait 275px

## Карусель 1: "Это вы ещё не видели"

### Что показывает
Новости, которые:
1. Совпадают с тегами пользователя (`matched_tags && user_tags`)
2. Пользователь **ещё не видел** (нет в `user_news_reads`)
3. Опубликованы за последние 90 дней

### Сортировка
Новые сверху (`published_at DESC`)

### API
```
GET /api/news?limit=50
```

### Код компонента
`UnreadNewsCarousel.tsx`

### Explicit Read Only

Новость **НЕ** считается прочитанной при скролле. Отметка "прочитано" срабатывает только по одному из трёх условий:

| Способ | Механика |
|--------|----------|
| (а) Клик на карточку | `onClick` → `markAsRead.mutate(id)` |
| (б) Кнопка ✓ | Явное действие пользователя |
| (в) 2 секунды в viewport | `IntersectionObserver` при 80%+ видимости (`threshold: 0.8`), таймер 2000ms |

**Визуальная обратная связь при отметке:**
- Карточка становится **полупрозрачной**
- Появляется **зелёная рамка** — индикатор "прочитано"
- `fade-out: 900ms` — плавное исчезновение
- Через `REMOVE_DELAY: 180_000ms` (3 минуты) — удаление из карусели

```
Простой скролл
    │
    ▼
┌─────────────────────┐
│  НЕ отмечает        │
│  как прочитанное    │
└─────────────────────┘
```

### Optimistic Updates

При `markAsRead` карточка **мгновенно переносится** из карусели 1 в карусель 2, без ожидания ответа API.

```tsx
// Реализация через queryClient.setQueryData()
queryClient.setQueryData(['unreadNews'], (old: News[]) =>
  old.filter(n => n.id !== newsId)
);
queryClient.setQueryData(['readNews'], (old: News[] = []) =>
  [news, ...old]
);
// Затем фоном — реальный POST /api/news/:id/read
```

**Эффект:** пользователь видит результат сразу, без задержки сети. Карточка плавно исчезает из первой карусели и появляется во второй.

### Tag Reactivity

При добавлении/удалении тега карусель **автоматически перезагружается**:

```
Изменение тегов
    │
    ▼
tagVersion++ в AuthContext
    │
    ▼
queryClient.invalidateQueries({ queryKey: ['unreadNews'] })
    │
    ▼
Карусель перезагружается с новым набором тегов
```

### Удаление прочитанных

| Параметр | Значение | Описание |
|----------|----------|----------|
| `REMOVE_DELAY` | `180_000` | 3 минуты — через сколько карточка удаляется из "непрочитанных" |
| `fade-out` | `900ms` | Длительность анимации исчезновения |

**Визуальные эффекты перед удалением:**
- Карточка становится **полупрозрачной** (`opacity` снижается)
- Появляется **зелёная рамка** — индикатор "отмечено как прочитанное"
- Через 3 минуты — полное исчезновение из карусели 1

```
Пользователь кликнул / нажал ✓
    │
    ▼
┌─────────────────────┐
│  Карточка:          │
│  • полупрозрачная   │
│  • зелёная рамка    │
│  • fade-out 900ms   │
└──────────┬──────────┘
           │
           ▼
    ┌──────────────┐
    │  REMOVE_DELAY│
    │   3 минуты   │
    └──────┬───────┘
           │
           ▼
    Карточка удалена
    из "непрочитанных"
```

## Карусель 2: "Вся лента"

### Что показывает
Новости, которые:
1. Совпадают с тегами пользователя (`matched_tags && user_tags`)
2. Пользователь **уже видел** (есть в `user_news_reads`)
3. Опубликованы за последние 90 дней

### Сортировка
Новые сверху (`published_at DESC`) — новые прочитанные первыми

### API
```
GET /api/news?history=true&limit=50
```

### Код компонента
`AllNewsCarousel.tsx`

### Fade-in animation

Когда новость переходит из карусели 1 в карусель 2 через **optimistic update**, она появляется с анимацией:

```css
@keyframes fadeInSlide {
  from {
    opacity: 0;
    transform: translateY(12px) scale(0.97);
  }
  to {
    opacity: 1;
    transform: translateY(0) scale(1);
  }
}

.carousel-2-card-enter {
  animation: fadeInSlide 400ms ease-out;
}
```

> Optimistic update = мгновенный перенос без ожидания ответа API

### Логика
Это **история просмотров**. Пользователь может вернуться к уже прочитанным новостям. Первая карусель показывает новое — вторая показывает что уже было просмотрено.

## Карусель 3: "Общая лента"

### Что показывает
Все новости из базы данных, **у которых есть хотя бы один тег**:
1. **Без фильтра по пользовательским тегам** — любые тегированные новости
2. **Новости без тегов исключаются** — они бесполезны и несут нагрузку
3. Опубликованы за последние 90 дней

### Сортировка
Новые сверху (`published_at DESC`)

### API
```
GET /api/news?global=true&limit=50
```

### Код компонента
`GlobalNewsCarousel.tsx`

### Логика
Видна **всегда и без логина**. Даже если у пользователя нет тегов или он не авторизован — он видит общую ленту всех новостей. Это даёт представление о том, какие новости собирает система.

## Sentiment Analysis

**Трёхуровневая система:**

### Level 1: Keyword-based (быстрый, без API)

| Тональность | Ключевые слова |
|-------------|---------------|
| Positive | рост, прибыль, рекорд, превысил, успех, повышение, рали |
| Negative | падение, убыток, кризис, снижение, крах, санкции |
| Neutral | нет ключевых слов — по умолчанию |

### Level 2: LLM Sentiment (Kimi API)

Выполняется когда есть `KIMI_API_KEY`:

```
analyzeSentimentLLM(title, summary) → 'positive' | 'negative' | 'neutral'
```

| Поле | Значение |
|------|----------|
| `sentiment` | `'positive'` \| `'negative'` \| `'neutral'` |
| `sentiment_source` | `'llm'` — если определено через Kimi, `'keyword'` — если fallback |

### Level 3: Tag Impact (Kimi API)

```
analyzeTagImpact(title, summary, tags) → TagImpact[]
```

**Формат ответа:**
```ts
interface TagImpact {
  tag: string;
  impact: 'positive' | 'negative' | 'neutral';
  reasoning: string;
}
```

- `tag_impact` хранится в БД как **JSONB** (`tag_impact JSONB DEFAULT '[]'`)
- Позволяет показать, какие именно теги повлияли на тональность новости

## Liquid Glass UI

Карточки новостей используют эффект **"liquid glass"**:

```css
.news-card {
  backdrop-filter: blur(16px);
  border: 1px solid var(--sentiment-color, rgba(156, 163, 175, 0.3));
  box-shadow: 0 4px 20px -4px var(--sentiment-glow);
}
```

**3 цветовых варианта** в зависимости от sentiment:

| Sentiment | Цвет glow | CSS-переменная |
|-----------|----------|---------------|
| Positive | Зелёный | `rgba(52, 211, 153)` |
| Negative | Красный | `rgba(248, 113, 113)` |
| Neutral | Серый | `rgba(156, 163, 175)` |

**Эффекты:**
- `backdrop-filter: blur(16px)` — матовое стекло
- Градиентная рамка в цвет sentiment
- Glow shadow (`0 4px 20px -4px`) в цвет sentiment

## User-Defined Tags

Пользователь может создать **собственный тег** через поиск:

### Flow создания

```
Пользователь вводит название тега
    │
    ▼
Dropdown: "Создать тег 'X'"
    │
    ▼
POST /api/user/tags/custom
    │
    ▼
Backend:
  1. Генерирует keywords (tagName + транслит + склонения)
  2. Создаёт запись в user_defined_tags
  3. BACKFILL: сканирует ВСЕ существующие новости,
     ищет совпадения по keywords
  4. Обновляет matched_tags для найденных статей
    │
    ▼
Frontend: tagVersion++ → invalidateQueries
    │
    ▼
Карусели перезагружаются
    │
    ▼
Новый тег сразу появляется в ленте (если есть совпадения)
```

### Реактивность

```
tagVersion++ в AuthContext
    │
    ├──→ invalidateQueries(['unreadNews'])
    ├──→ invalidateQueries(['readNews'])
    └──→ invalidateQueries(['globalNews'])
```

## Tag Impact Pills

На карточках отображаются **цветные индикаторы тегов** — "pills" с цветовым кодированием влияния:

| Влияние | Цвет | Пример |
|---------|------|--------|
| `positive` | 🟢 Зелёный | `акции ↑` |
| `negative` | 🔴 Красный | `убыток ↓` |
| `neutral` | ⚪ Серый | `отчёт` |

```tsx
// Рендер pills на карточке
{news.tag_impact?.map((ti) => (
  <span
    key={ti.tag}
    className={cn(
      "px-2 py-0.5 rounded-full text-xs font-medium",
      ti.impact === 'positive' && "bg-green-500/20 text-green-400 border border-green-500/30",
      ti.impact === 'negative' && "bg-red-500/20 text-red-400 border border-red-500/30",
      ti.impact === 'neutral'  && "bg-gray-500/20 text-gray-400 border border-gray-500/30",
    )}
  >
    {ti.tag}
  </span>
))}
```
