## Общий принцип

| # | Название | Что показывает | Для кого | Sentiment-карточки | Статус |
|---|----------|---------------|----------|-------------------|--------|
| 1 | **"Это вы ещё не видели"** | Непрочитанные новости по тегам пользователя | Только залогиненным | Liquid glass + Tag Impact pills | ✅ Работает |
| 2 | **"Вся лента"** | Прочитанные новости по тегам (хронология) | Только залогиненным | Liquid glass + Tag Impact pills | ✅ Работает |
| 3 | **"Общая лента"** | Все новости за 90 дней (с тегами и без) | Всем, включая без логина | Liquid glass + Tag Impact pills | ✅ Работает |

**Логика:**
- 1-я карусель = **новое** (ещё не видел) → landscape 425×225px
- 2-я карусель = **история** (уже видел, можно вернуться) → portrait 275px
- 3-я карусель = **полная лента** (всё что есть в базе) → portrait 275px

## Карусель 1: "Это вы ещё не видели"

### Что показывает
Новости, которые:
1. Совпадают с тегами пользователя (`matched_tags && user_tags`)
2. Пользователь **ещё не видел** (нет в `user_news_reads`)
3. Опубликованы за последние 90 дней

### Сортировка
Новые сверху (`published_at DESC`)

### API
```
GET /api/news?limit=50
```

### Код компонента
`UnreadNewsCarousel.tsx`

### Explicit Read Only

Новость **НЕ** считается прочитанной при скролле. Отметка "прочитано" срабатывает только по одному из трёх условий:

| Способ | Механика |
|--------|----------|
| (а) Клик на карточку | `onClick` → `markAsRead.mutate(id)` |
| (б) Кнопка ✓ | Явное действие пользователя |
| (в) 2 секунды в viewport | `IntersectionObserver` при 80%+ видимости (`threshold: 0.8`), таймер 2000ms |

**Визуальная обратная связь при отметке:**
- Карточка становится **полупрозрачной** (`opacity: 0.5`)
- Появляется **зелёная рамка** `2px solid rgba(52, 211, 153, 0.4)` — индикатор "прочитано"
- `fade-out: 900ms` — плавное исчезновение
- Через `REMOVE_DELAY: 180_000ms` (3 минуты) — удаление из карусели

```
Простой скролл
    │
    ▼
┌─────────────────────┐
│  НЕ отмечает        │
│  как прочитанное    │
└─────────────────────┘
```

### Optimistic Updates

При `markAsRead` карточка **мгновенно переносится** из карусели 1 в карусель 2, без ожидания ответа API.

```tsx
// Реализация через queryClient.setQueryData()
queryClient.setQueryData(['unreadNews'], (old: News[]) =>
  old.filter(n => n.id !== newsId)
);
queryClient.setQueryData(['historyNews'], (old: News[] = []) =>
  [news, ...old]
);
// Затем фоном — реальный POST /api/news/:id/read
```

**Эффект:** пользователь видит результат сразу, без задержки сети. Карточка плавно исчезает из первой карусели и появляется во второй.

### Tag Reactivity

При добавлении/удалении тега карусель **автоматически перезагружается**:

```
Изменение тегов
    │
    ▼
tagVersion++ в AuthContext
    │
    ▼
queryClient.invalidateQueries({ queryKey: ['unreadNews'] })
    │
    ▼
Карусель перезагружается с новым набором тегов
```

### Удаление прочитанных

| Параметр | Значение | Описание |
|----------|----------|----------|
| `REMOVE_DELAY` | `180_000` | 3 минуты — через сколько карточка удаляется из "непрочитанных" |
| `fade-out` | `900ms` | Длительность анимации исчезновения |

**Визуальные эффекты перед удалением:**
- Карточка становится **полупрозрачной** (`opacity: 0.5`)
- Появляется **зелёная рамка** `2px solid rgba(52, 211, 153, 0.4)` — индикатор "отмечено как прочитанное"
- Через 3 минуты — полное исчезновение из карусели 1

```
Пользователь кликнул / нажал ✓
    │
    ▼
┌─────────────────────┐
│  Карточка:          │
│  • opacity: 0.5     │
│  • зелёная рамка    │
│    rgba(52,211,153) │
│  • fade-out 900ms   │
└──────────┬──────────┘
           │
           ▼
    ┌──────────────┐
    │  REMOVE_DELAY│
    │   3 минуты   │
    └──────┬───────┘
           │
           ▼
    Карточка удалена
    из "непрочитанных"
```

## Карусель 2: "Вся лента"

### Что показывает
Новости, которые:
1. Совпадают с тегами пользователя (`matched_tags && user_tags`)
2. Пользователь **уже видел** (есть в `user_news_reads`)
3. Опубликованы за последние 90 дней

### Сортировка
Новые сверху (`published_at DESC`) — новые прочитанные первыми

### API
```
GET /api/news?history=true&limit=50&page=N
```

Ответ:
```json
{
  "articles": [...],
  "total": 1234,
  "page": 1,
  "hasMore": true
}
```

### Бесконечный скролл
Карусель подгружает историю порциями по 50 штук:
- Используется `useInfiniteQuery` с `queryKey: ['historyNews']`
- `initialPageParam: 1`
- `getNextPageParam` берёт следующую страницу, пока `hasMore === true`
- В конце трека стоит невидимый `sentinel`, за которым следит `IntersectionObserver`
- Когда sentinel входит в viewport — вызывается `fetchNextPage()`
- Подгруженные страницы склеиваются через `pages.flatMap(p => p.articles)`

### Код компонента
`AllNewsCarousel.tsx`

### Анимация появления новых карточек (Frost Appear + FLIP)

Когда новость переходит из карусели 1 в карусель 2 через **optimistic update** или приходит SSE-refresh:

1. **Frost Appear** — новая карточка материализуется из тёмного тумана за 1.8s:
   - `opacity: 0 → 1`
   - `scale: 0.97 → 1`
   - `brightness: 0.3 → 1`
   - `blur: 2px → 0`
   - Тёмный frosted-glass overlay (`backdrop-filter: blur(20px)`) растворяется.
2. **FLIP-сдвиг** — старые видимые карточки плавно сдвигаются вправо (0.8s), чтобы освободить место новой.
3. **Каскад** — первые 5 новых карточек в батче появляются с задержкой 0.15s друг за другом.

```css
.news-appear-wrapper {
  position: relative;
  opacity: 0;
  animation: newsCardAppear 1.8s cubic-bezier(0.25, 0.1, 0.25, 1) forwards;
}

.news-frost-layer {
  position: absolute;
  inset: 0;
  border-radius: 0.75rem;
  background: linear-gradient(
    180deg,
    rgba(6, 6, 6, 0) 0%,
    rgba(6, 6, 6, 0.5) 30%,
    rgba(6, 6, 6, 0.9) 70%,
    rgba(6, 6, 6, 0.95) 100%
  );
  backdrop-filter: blur(20px);
  -webkit-backdrop-filter: blur(20px);
  pointer-events: none;
  z-index: 2;
  animation: newsFrostLift 1.8s cubic-bezier(0.4, 0, 0.2, 1) forwards;
}

.flip-card-shifting {
  will-change: transform;
}
```

**Хук:** `useFlipAnimation.ts` — отслеживает новые ID, запускает FLIP только если новые элементы добавлены в начало списка и пользователь не скроллил вправо (`scrollLeft <= 50`). При infinite scroll сдвиг не запускается.

> Optimistic update = мгновенный перенос без ожидания ответа API

### Логика
Это **история просмотров**. Пользователь может вернуться к уже прочитанным новостям. Первая карусель показывает новое — вторая показывает что уже было просмотрено.

## Карусель 3: "Общая лента"

### Что показывает
Все новости из базы данных:
1. **Без фильтра по пользовательским тегам** — любые новости
2. Опубликованы за последние 90 дней

### Сортировка
Новые сверху (`published_at DESC`)

### API
```
GET /api/news/global?limit=50&page=N
```

Ответ:
```json
{
  "articles": [...],
  "total": 1234,
  "page": 1,
  "hasMore": true
}
```

### Бесконечный скролл
Аналогично карусели 2 — порции по 50 через `useInfiniteQuery`:
- `queryKey: ['globalNews']`
- `sentinel` + `IntersectionObserver` внутри скролл-контейнера
- Автоподгрузка при приближении к концу ленты

### Код компонента
`GlobalNewsCarousel.tsx`

### Логика
Видна **всегда и без логина**. Даже если у пользователя нет тегов или он не авторизован — он видит общую ленту всех новостей. Это даёт представление о том, какие новости собирает система.

### Real-time обновление
Общая лента обновляется в реальном времени через SSE:
- `useSseNews.ts` подключается к `GET /api/news/stream` для всех пользователей.
- При `event: refresh` от backend вызывается `refetchQueries({ queryKey: ['globalNews'] })`.
- Новые статьи появляются в карусели сразу после завершения цикла `NewsSourceManager`.

### Анимация появления новых карточек (Frost Appear + FLIP)

Аналогично карусели 2: новые статьи появляются с анимацией Frost Appear (1.8s), а старые видимые карточки плавно сдвигаются вправо через FLIP (0.8s). За анимацию отвечает тот же хук `useFlipAnimation.ts`.

- Новые карточки оборачиваются в `.news-appear-wrapper` + `.news-frost-layer`.
- Старые карточки получают класс `.flip-card-shifting` на время FLIP-сдвига.
- При infinite scroll (`fetchNextPage`) новые карточки в конце появляются без FLIP-сдвига.

## Sentiment Analysis

**Трёхуровневая система:**

### Level 1: Keyword-based (быстрый, без API)

| Тональность | Ключевые слова |
|-------------|---------------|
| Positive | рост, прибыль, рекорд, превысил, успех, повышение, рали |
| Negative | падение, убыток, кризис, снижение, крах, санкции |
| Neutral | нет ключевых слов — по умолчанию |

### Level 2: LLM Sentiment (Kimi API)

Выполняется когда есть `KIMI_API_KEY`:

```
analyzeSentimentLLM(title, summary) → 'positive' | 'negative' | 'neutral'
```

| Поле | Значение |
|------|----------|
| `sentiment` | `'positive'` \| `'negative'` \| `'neutral'` |
| `sentiment_source` | `'llm'` — если определено через Kimi, `'keyword'` — если fallback |

### Level 3: Tag Impact (Kimi API)

```
analyzeTagImpact(title, summary, tags) → TagImpact[]
```

**Формат ответа:**
```ts
interface TagImpact {
  tag: string;
  impact: 'positive' | 'negative' | 'neutral';
  reasoning: string;
}
```

- `tag_impact` хранится в БД как **JSONB** (`tag_impact JSONB DEFAULT '[]'`)
- Позволяет показать, какие именно теги повлияли на тональность новости

## Liquid Glass UI

Карточки новостей используют эффект **"liquid glass"** — полупрозрачное стекло с цветовым glow в зависимости от sentiment:

### Базовый CSS

```css
.news-card {
  /* Фон: градиент с sentiment-цветом */
  background: config.glassBg;
  /* Градиент: от sentiment-цвета 0.06 → 0.02 */

  /* Рамка */
  border: 1px solid config.glassBorder;
  /* opacity 0.15 для positive/negative, 0.08 для neutral */

  /* Тень */
  box-shadow: config.glowShadow;
  /* 0 4px 20px -4px sentiment-color(0.15) */

  /* Размытие */
  backdrop-filter: blur(12px) saturate(180%);

  /* Скругление */
  border-radius: 12px; /* rounded-xl */
}
```

### Hover-эффект

```css
.news-card:hover {
  border-color: config.glassBorderHover;
  /* opacity 0.35 — рамка ярче при наведении */

  box-shadow: config.glowShadowHover;
  /* тень усиливается */
}
```

### Декоративные линии

```css
/* Верхняя линия — градиент (transparent → sentiment → transparent) */
.news-card::before {
  content: '';
  position: absolute;
  top: 0;
  left: 10%;
  right: 10%;
  height: 1px;
  background: linear-gradient(
    90deg,
    transparent,
    var(--sentiment-color),
    transparent
  );
  opacity: 0.60;
}

/* Нижний glow — 2px градиентная линия */
.news-card::after {
  content: '';
  position: absolute;
  bottom: 0;
  left: 20%;
  right: 20%;
  height: 2px;
  background: linear-gradient(
    90deg,
    transparent,
    var(--sentiment-color),
    transparent
  );
  opacity: 0.40;
}

.news-card:hover::after {
  opacity: 0.70; /* glow усиливается на hover */
}
```

### Цветовая конфигурация по sentiment

| Параметр | Positive | Negative | Neutral |
|----------|----------|----------|---------|
| `color` | `#34D399` | `#EF4444` | `#9CA3AF` |
| `glassBg` | `linear-gradient(180deg, rgba(52,211,153,0.06), rgba(52,211,153,0.02))` | `linear-gradient(180deg, rgba(239,68,68,0.06), rgba(239,68,68,0.02))` | `linear-gradient(180deg, rgba(255,255,255,0.03), rgba(255,255,255,0.01))` |
| `glassBorder` | `rgba(52,211,153,0.15)` | `rgba(239,68,68,0.15)` | `rgba(255,255,255,0.08)` |
| `glassBorderHover` | `rgba(52,211,153,0.35)` | `rgba(239,68,68,0.35)` | `rgba(255,255,255,0.20)` |
| `glowShadow` | `0 4px 20px -4px rgba(52,211,153,0.15)` | `0 4px 20px -4px rgba(239,68,68,0.15)` | `0 2px 12px -4px rgba(0,0,0,0.30)` |
| `glowShadowHover` | усиленная зелёная тень | усиленная красная тень | усиленная тень |

**Эффекты:**
- `backdrop-filter: blur(12px) saturate(180%)` — матовое стекло с насыщенностью
- Градиентная верхняя линия в цвет sentiment (opacity 0.60)
- Нижний glow (2px, opacity 0.40 → 0.70 на hover)
- Градиентный фон: sentiment-цвет 0.06 → 0.02 сверху вниз
- Рамка с opacity 0.15 (0.08 для neutral), на hover → 0.35

### Sentiment Score Badge (v7.11+)

Плашка сантимента отображает **оценку от LLM** (−10..+10):

```
Было: [↑] Позитив          Стало: [↑] Позитив +5
       [↓] Негатив                [↓] Негатив -3
```

**Цветовая шкала score:**
| Score | Sentiment | Цвет | Пример badge |
|-------|-----------|------|-------------|
| −10..−5 | negative | 🔴 Красный | `[↓] Негатив -8` |
| −4..−1 | negative | 🔴 Красный | `[↓] Негатив -2` |
| 0 | neutral | ⚪ Серый | `[→] Нейтрально 0` |
| +1..+4 | positive | 🟢 Зелёный | `[↑] Позитив +3` |
| +5..+10 | positive | 🟢 Зелёный | `[↑] Позитив +7` |

**Tooltip с reasoning (v7.12+):** При наведении на плашку — всплывает `title` с 2 абзацами объяснения от LLM:
```

```

## User-Defined Tags

Пользователь может создать **собственный тег** через поиск:

### Flow создания

```
Пользователь вводит название тега
    │
    ▼
Поиск /api/tags/search (debounce 200ms) → searchResults
    │
    ├─── Точное совпадение по tag_name найдено?
    │     └── Да: handleSelectSuggestion(existingTag) → POST /api/user/tags
    │         (подписка на существующий тег, без создания дубля)
    │
    └─── Нет совпадения
          │
          ▼
    Enter / "Создать тег 'X'"
          │
          ▼
    POST /api/user/tags { tagId, tagName, tagType: 'auto' }
          │
          ▼
    Backend createUserTag():
      1. Ищет тег по tag_id ИЛИ LOWER(tag_name)
      2. Если найден — подписывает на существующий tag_id
      3. Если не найден — LLM enrichment, keywords, INSERT INTO user_defined_tags
      4. INSERT INTO portfolios ON CONFLICT DO NOTHING
          │
          ▼
    Frontend: tagVersion++ → invalidateQueries
          │
          ▼
    Карусели перезагружаются
          │
          ▼
    Новый тег появляется в ленте через RSS pipeline
    (ручной backfill по всей базе — POST /api/user/tags/custom)
```

### Реактивность

```
tagVersion++ в AuthContext
    │
    ├──→ invalidateQueries(['unreadNews'])
    ├──→ invalidateQueries(['historyNews'])
    └──→ invalidateQueries(['globalNews'])
```

## ⚠️ КРИТИЧЕСКОЕ РАЗГРАНИЧЕНИЕ: NewsFeed vs Карусели

> **Этот раздел существует, чтобы предотвратить повторение инцидента (см. историю ниже).**
>
> **ПРАВИЛО: NewsFeed и Карусели — это ДВЕ РАЗНЫЕ фичи. Никогда не смешивать их код, API и навигацию.**

### Что есть что

| Аспект | **NewsFeed** (/news) | **Карусели** (Home.tsx) |
|--------|----------------------|-------------------------|
| **Тип** | Отдельная страница | Компоненты на главной |
| **URL** | `/news` — через React Router | Нет отдельного URL, часть `/` |
| **Открытие** | Клик на теге → `navigate('/news')` | Выбор тега в портфолио → фильтр внутри Home |
| **Вид** | Вертикальный список статей | 3 горизонтальные прокрутки |
| **API** | `GET /api/news?all=true`, `GET /api/news/search`, `GET /api/news/tags/:tagId`, `POST /api/news/:id/read` | `GET /api/news`, `?history=true`, `/api/news/global` |
| **Компоненты** | `NewsFeed.tsx`, `NewsDetailModal.tsx` | `UnreadNewsCarousel.tsx`, `AllNewsCarousel.tsx`, `GlobalNewsCarousel.tsx` |
| **Клик по карточке** | Открывает `NewsDetailModal` (детали + теги + сантимент) | Открывает `NewsDetailModal` (тот же компонент) |
| **Назначение** | Подробный просмотр новостей по тегу | Быстрый обзор на главной |

### История инцидента (2026-05-30)

**Что произошло:**
Попытка добавить `TagEnrichment` — информацию о том, почему новость попала в ленту (matching logic). Реализация перепутала NewsFeed с каруселями и начала менять:
- `NewsFeed.tsx` — добавлялась логика, которой там не должно быть
- API endpoint `/api/news` — начали добавлять tag filter, который ломал карусели
- Auth requirements — меняли public → auth → public, ломая и карусели, и фид
- Frontend бандл — рос за счёт нового компонента, дублировал логику

**Последствия:**
- Карусели перестали загружаться
- NewsFeed показывал пустоту
- Auth токены сбрасывались на 401
- Frontend не деплоился из-за TS ошибок

**Решение:**
- Frontend откачен на коммит `6b707ce` (v7.11.1)
- Backend откачен на коммит `76c0f8a` (v7.9)
- TagEnrichment полностью удалён из кодовой базы

**Вывод:**
> **NewsFeed и Карусели — изолированные системы. Изменения в одной НЕ должны затрагивать другую. Если нужна новая фича, она либо встраивается в существующую систему без изменения другой, либо создаётся как третья независимая система.**

### Если в будущем нужен Tag Matching Logic

**Правильный подход:**

```
Вариант А: Tooltip/Modal при наведении на тег в карточке
  → Компонент внутри карусели, не затрагивает NewsFeed
  → Данные приходят с существующим tag_impact

Вариант Б: Отдельная страница /tag/:tagName/explain
  → Свой роут, свой компонент, свой API
  → НЕ модифицирует NewsFeed.tsx и не трогает карусели

Вариант В: Расширение существующего NewsFeed
  → ТОЛЬКО внутри NewsFeed.tsx
  → API: ТОЛЬКО новый endpoint, НЕ трогать /api/news
```
