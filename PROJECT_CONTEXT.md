## 1. VISION

### Что такое PULSE
PULSE — русскоязычная платформа для инвесторов, которая отслеживает новости по портфелю пользователя и отправляет еженедельный репорт вместо бесконечного скролла ленты.

### Продуктовая ценность
> **От 2 часов чтения новостей → 5 минут репорта**

| Без PULSE | С PULSE |
|-----------|---------|
| 200+ новостей в день | 10-20 релевантных |
| Читать всё самому | Итоговый репорт |
| Пропускаешь важное | Ничего не пропустишь |
| 2 часа на чтение | 5 минут на репорт |

### Продуктовый круг (замкнутый)
```
Пользователь задаёт теги (свой портфель)
  ↓
PULSE следит за этими компаниями 24/7 (32 источника)
  ↓
Отбираем ТОЛЬКО релевантные новости
  ↓
Анализируем sentiment (позитив / нейтрально / негатив)
  ↓
Каждую неделю — репорт в Telegram + Email
  ↓
Пользователь читает 2 абзаца вместо 200 новостей
  ↓
Понедельник — новая неделя, новый цикл
```

### Elevator Pitch
> "Слежу за портфелем → читаю все → отбираю релевантное → анализирую → еженедельный репорт → 5 минут вместо 2 часов"

## 2. ЦЕЛЕВАЯ АУДИТОРИЯ

### Primary: Частный инвестор (Россия)
- **Портфель:** 10-50 позиций (акции РФ + США)
- **Проблема:** Не хватает времени следить за всеми новостями
- **Потребность:** Быстро понимать, что происходит с его компаниями
- **Поведение:** Проверяет телефон утром, в обед, вечером
- **Канал:** Telegram (приоритет #1), Email (#2)

### Secondary: Начинающий инвестор
- **Портфель:** 3-10 позиций
- **Проблема:** Не понимает, какие новости важны
- **Потребность:** Простое понимание sentiment по компаниям
- **Поведение:** Бесплатный тариф, лента на сайте

### Anti-persona
- Дневные трейдеры (им нужна скорость, не агрегация)
- Институционалы (у них есть Bloomberg)
- Пассивные инвесторы ETF (им не нужен трекинг отдельных компаний)

## 3. СЦЕНАРИИ ИСПОЛЬЗОВАНИЯ

### UC-1: Первый визит (гость)
1. Открывает сайт → видит Hero "Отслеживай все"
2. Видит поиск → пробует ввести компанию
3. Видит подсказку "Войдите, чтобы добавлять теги"
4. Нажимает "Начать" → регистрация
5. Попадает в приложение → добавляет 3 тега (лимит Free)
6. Видит новости по тегам

### UC-2: Ежедневное использование (авторизован)
1. Открывает сайт → видит свои теги
2. Видит новости по портфелю
3. Читает ленту → кликает на тег → фильтр по тегу
4. Вечером — получает Telegram-алерт о резком сдвиге sentiment

### UC-3: Получение репорта (Premium)
1. Воскресенье 13:00 МСК
2. Telegram-бот присылает еженедельный дайджест
3. Группировка по тегам, sentiment-анализ, итоговая статистика
4. 5 минут чтения вместо 200 новостей

### UC-4: Управление подпиской
1. Заходит в профиль → видит "Осталось N дней"
2. Нажимает "Продлить" → скользящая скидка
3. Оплачивает через ЮKassa → Premium продлён

### UC-5: Настройка уведомлений
1. Профиль → вкладка "Уведомления"
2. Выбирает: каналы, частота, тип, алерты, время, тихие часы, формат, язык
3. Сохраняет → настройки применяются

## 5. АРХИТЕКТУРА

### Frontend
- **URL:** https://pulse-frontend-jt53.onrender.com
- **Stack:** React 19 + TypeScript + Vite + Tailwind CSS v3.4 + shadcn/ui + Framer Motion
- **Router:** HashRouter (SPA)
- **Auth:** JWT в localStorage, API-only (useAuth → бэкенд)

### Backend
- **URL:** https://pulse-api-bsov.onrender.com
- **Stack:** Node.js 20 + Express + TypeScript
- **DB:** SQLite (sql.js/WASM) — dev / PostgreSQL — prod
- **Auth:** JWT (bcryptjs, jsonwebtoken)
- **RSS:** 32 источника (13 RU + 19 EN), cron каждые 15 мин
- **Translate:** 3-уровневая — cache → Kimi API → Google Translate
- **Reports:** Воскресенье 13:00 МСК, Telegram Bot + SendGrid

### Инфраструктура
- **Frontend:** Render Static Site (автодеплой из GitHub)
- **Backend:** Render Web Service (Docker, автодеплой из GitHub)
- **Git:** `vladfa2010/pulse-frontend` + `vladfa2010/pulse`

## 6. КРИТИЧЕСКИЕ ПРАВИЛА

### Продуктовые
1. **Email = PRIMARY KEY**, не меняется после регистрации
2. **Все EN новости → RU перевод**, пользователь видит только русский
3. **Мок-новости ЗАПРЕЩЕНЫ** — только реальные RSS
4. **Репорты только Premium** — ключевое отличие тарифов
5. **Все платежи сохраняются** в таблице `payments`

### Технические
6. **localStorage = только JWT токен**, не база данных
7. **useAuth = только через API**, никакого hardcoded demo
8. **Git commit после КАЖДОЙ доработки**
9. **Обновлять оба репозитория синхронно**
10. **Render:** `base: '/'`, **GitHub Pages:** `base: '/pulse-frontend/'`
